What is OAuth?

OAuth (Open Authorization) allows third-party applications to access user data without requiring users to share their passwords.

Key Terminologies

  • Client: The application requesting access to resources.
  • Resource Owner: The user who possesses the resources.
  • Resource: Data or images exposed through APIs, etc.
  • Resource Server: The server that hosts protected resources.
  • Authorization Server: The server that authorizes clients and issues access tokens.
  • User-Agent: The browser or application used for communication with the authorization server.
  • Access Token: A token granted for accessing resources after successful authorization.
  • Refresh Token: A token used to renew the access token.

For example, a user can allow a printing service to access their photos stored on a photo-sharing service without needing to share their login details.

OAuth 2.0 Flows

  1. Authorization Code Grant
  2. Authorization Code Grant with PKCE
  3. Client Credentials Grant
  4. Resource Owner Password Credentials Grant
  5. Implicit Grant

1. Authorization Code Grant

This is a browser-based flow designed for web and mobile applications where the user is redirected to the authorization endpoint to obtain an authorization code.

2. Authorization Code Grant with PKCE

This flow is tailored for public clients, leveraging Proof Key for Code Exchange (PKCE) to generate a secret at runtime for enhanced security during authorization.

2.1. Transformations - Understanding SHA256

A random string, known as the Code Verifier, is converted into a Code Challenge using the SHA256 algorithm.

3. Client Credentials Grant

This grant type is used for machine-to-machine communications where user consent is not a requirement.

4. Resource Owner Password Credentials Grant

This flow should only be used when there is a high degree of trust between the resource owner and the client application.

5. Implicit Grant

This flow is no longer recommended by standards organizations! The access token is transmitted in the URL fragment, which introduces security vulnerabilities.
