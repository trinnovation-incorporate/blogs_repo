What's OAuth?

OAuth (Open Authorization) enables third-party websites or apps to access user's data without requiring them to share their credentials.

Terminologies

  • Client: The application seeking access to resources.
  • Resource Owner: The user who owns the resources.
  • Resource: Images, data exposed via APIs, etc.
  • Resource Server: Server hosting protected resources.
  • Authorization Server: Server authorizing clients and issuing access tokens.
  • User-Agent: The browser or app used to communicate with the authorization server.
  • Access Token: A token for accessing resources, issued after successful authorization.
  • Refresh Token: A token used to replenish the access token.

Example: A user grants a printing service access to their photo hosted on a photo-sharing service without sharing credentials.

OAuth 2.0 Flows

  1. Authorization Code Grant
  2. Authorization Code Grant with PKCE
  3. Client Credentials Grant
  4. Resource Owner Password Credentials Grant
  5. Implicit Grant

1. Authorization Code Grant

A browser-based flow for web and mobile apps, where the user is redirected to the authorization endpoint to get the authorization code.

2. Authorization Code Grant with PKCE

For public clients, using Proof Key for Code Exchange (PKCE) to generate a secret at runtime for secure authorization.

2.1. Transformations - Meet SHA256

A random string (Code Verifier) is transformed into a Code Challenge using SHA256.

3. Client Credentials Grant

Used for machine-to-machine communication where user consent is not needed.

4. Resource Owner Password Credentials Grant

This flow is to be used only when there is a high degree of trust between the resource owner and the client.

5. Implicit Grant

This flow is no longer recommended officially! The access token is passed in the URL fragment, posing security risks.
